– csc accesses the iaas through portal with appropriate security mechanism and

post-conditions 
(optional)   
description – csc accesses and queries the csp portal to retrieve the list of supported network 
functions (e.g., ip address, vlan, virtual switch, load balance, firewall). 
– csc selects the appropriate network service from the query results and requests csp 
to create a network based on the selection. 
– csc requests the csp to attach the created network with the related virtual machine. 
– csc manages and monitors the created network during its lifecycle. it includes, but not 
limited to: 
• bind the network connectivity service to the related virtual machine;  
• query the network and its status; 
• update, upgrade or scaling the network; 
• execute performance metrics; 
• manage the ip address pool; 
• manage network services, such as vlan, virtual switch, load balance, firewall, 
gateway;  
• manage tenants' network according to the sla objectives.
